Design Process

This page documents how I planned and developed my portfolio website. It includes wireframes, accessibility decisions, and project-by-project process work to explain how each design developed from early ideas to final outcomes.

Wireframes and Layout Planning

I began by sketching both desktop and mobile layouts. These wireframes helped me decide where to place the navigation, profile image, project previews, and main content sections before coding the final pages. Planning both screen sizes early helped me think about responsiveness from the start instead of treating mobile as an afterthought.

Accessibility Statement

I designed this site with accessibility in mind by using semantic HTML elements such as header, nav, main, section, article, figure, and footer. I added alt text to meaningful images, used clear heading levels, and wrote descriptive navigation labels so the site is easier to read and navigate.

I also checked contrast between text and background colors, kept font sizes readable, and used spacing and large click areas to improve usability. Through media queries, I made sure the layout adapts to smaller screens without requiring horizontal scrolling. These choices support WCAG 2.0 principles of perceivability, operability, and readability.

UTM Housing Platform UX Project

This project focused on a medium-fidelity housing platform prototype for UTM students. The main task tested whether users could find an existing booking and cancel it without external help. The work centered on usability, interface clarity, and better communication during a high-stakes action such as cancellation.

Decisions and Design Thinking

The main design decision in this project was to make the cancellation flow easy to complete without outside help. That meant organizing the path through login, booking management, order details, and cancellation in a way that felt direct. At the same time, testing showed that completing the task was not enough on its own. Users also needed visual confidence.

Feedback showed that some buttons were not prominent enough, some labels were unclear, and the status after cancellation needed stronger confirmation. Because of that, the thought process behind this project shifted from simple task completion to clearer hierarchy, better naming, and stronger next-step feedback. This project shaped how I think about UX writing, interface emphasis, and transactional design.

Graphic Design Poster: Astroworld Tour

This poster project focused on composition, typography, color, and public-space presentation. I designed a concert poster for Travis Scott’s Astroworld Tour and then placed it into a mock-up environment. This project influenced how I think about scale, image placement, and visual hierarchy in the portfolio.

Graphic Design Process Work

Decisions and Design Thinking

I built this poster around the idea of spectacle and scale. The diagonal title creates movement across the page, while the sky background and floating imagery push the design toward a surreal, atmospheric mood. I grouped the date information so it could still be read quickly even with the more expressive image composition.

The major design decision was balancing readability with visual drama. I wanted the poster to feel energetic and dreamlike, but it still needed to function as event communication. That is why the typography, contrast, and placement of information became central decisions. This project helped me think more carefully about how layout choices can carry meaning instead of just organizing content.

3D Modeling for Kitty’s Wonderland

This project involved modeling environmental assets in Autodesk Maya for a cozy cat exploration game. I developed the furniture through multiple stages, from basic blockouts to refined forms, UV mapping, texturing, and final rendering. I separated the process into bed and wardrobe sections so each workflow is easier to follow.

Bed Modeling Process

Bed Design Thinking

I started the bed with simple shapes so I could focus on proportion before detail. That decision made it easier to adjust the silhouette early, especially the headboard curve and the main mass of the structure. Once the large forms felt stable, I moved into separating the mattress, frame, and visible structural pieces.

The key thought process here was controlling complexity in stages. Instead of trying to solve shape, texture, and rendering all at once, I treated the bed as a sequence: blockout first, refinement next, UV layout after that, and final rendering last. That let me keep the asset readable and consistent with the cozy environment of the game.

Wardrobe Modeling Process

Wardrobe Design Thinking

For the wardrobe, I approached the design through structure first and detail second. I began with a basic rectangular form, then cut into the front faces to establish the cabinet doors. This made the asset easier to control and kept the proportions clean before adding smaller features.

The design decision here was to keep the object simple enough for the game environment, but not so plain that it lost material and dimensional interest. The door extrusion, handle detail, UV layout, and final texture work all came from that balance between simplicity and believability.

Fan Page Website Design: NF

This front-end project focused on HTML, CSS, and JavaScript. I designed the site around a dark, high-contrast theme and used a structured layout with a banner, featured album image, content cards, and interactive buttons. This project influenced the way I approached structure and simple interaction in my portfolio.

Decisions and Design Thinking

I wanted this site to match NF’s emotional and intense music, so I chose a dark theme, large album image, and strong contrast between content blocks and background. That decision helped create a more focused atmosphere and kept the page visually tied to the artist’s tone.

I also wanted the page to feel active rather than static, so I added JavaScript interactions through alerts, confirmation prompts, and text changes. The thought process behind this project was to connect mood, content, and interaction in one small front-end design. It taught me that even simple web pages benefit from a clear theme and purposeful interactivity.

Website Color Palette

The color theme of this portfolio is built around a warm neutral base with a bold orange-red accent, grounded by a deep navy for headings and structure. The palette balances professionalism with energy — the navy provides clarity and trust, while the accent orange draws attention to key interactive elements like buttons and links.
